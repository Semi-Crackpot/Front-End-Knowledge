# Vue-Router

### 路由之间怎么跳转？
1.  声明式（标签跳转）：``<router-link :to="path">``
2.  编程式（js跳转）：``this.$router.push(path)``

### router-view标签的作用是什么？
- 用来指定路由渲染的位置

### 怎么设置默认路由？
- 在routes路由数组中，添加一个对象，用来设置默认路由
    ```js
    {
        path: '/',
        // redirect重定向到我们想要的主页
        redirect: '/home'
    }

### router-link有什么属性？
1.  to：跳转路由
2.  tag：修改router-link的标签
3.  replace：跳转后不会留下history记录
4.  active-class：当路由匹配成功后，自动给当前元素添加的类

### 动态路由怎么实现？
1.  注册动态路由
    ```js
    {
        path: '/home/:userId',
        components: User
    }
2.  设置动态路由的跳转
    ```js
    <router-link :to="'/user/' + userId">
    this.$router.push('/user/' + this.userId)
3.  获取动态数据：``this.$route.params.userId``

### 路由懒加载
- 将不同路由对应的组件分割成不同的文件，当路由被访问的时候才加载对应组件
1.  正常加载：``import Home from ...``
2.  懒加载：``const Home = () => import(...)``

### 嵌套路由
    ```js
    routes: [
        {
            path: '/home',
            components: Home,
            children: [
                {
                    // 子路由的path会自动在父路由后添加/，再添加path的值
                    path: '',
                    redirect: 'news'
                },
                {
                    path: 'news',
                    components: News,
                }
            ]
        }
    ]
    ```

### 怎么给路由传递参数？
1.  跳转时传递query
    ```html
    <router-link :to="{path: '/home', query: {name: 'wyz', age: 18}}">
2.  获取query
    - this.$route.query
    - this.$route.query.name

### router和route的区别是什么？
1.  ``this.$router``：创建的Router对象实例
2.  ``this.$route``：当前匹配成功的路由

### 路由的导航守卫是什么?
1.  作用：监听路由跳转的过程
2.  导航守卫有几种钩子？
    - 全局前置钩子：beforeEach
        ```js
        // router是创建的Router实例对象
        router.beforeEach((to, from, next) => {
            // 修改页面标题为跳转的路由的元信息的title数据
            document.title = to.meta.title
            next()
        })
        // 设置每个路由的元信息
        {
            path: '/home',
            components: Home,
            meta: {
                title: '首页'
            }
        }
    - 全局后置钩子：afterEach，不需要执行next()
    - 路由独享钩子：beforeEnter，写在单个路由里
